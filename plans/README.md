## Findings considered and rejected

- Resample entire realtime stream to 48 kHz: rejected as default architecture. Adds buffering, latency, clock-drift handling, callback frame conversion, CPU, and failure modes while Core Audio provides device-bound tap format matching.
- Rely on AUHAL to convert 44.1↔48 kHz: rejected. Apple AUHAL guidance requires device/client sample-rate match; SRC needs explicit buffered conversion.
- Change physical device nominal rate to 48 kHz: rejected. Violates Airwave safety/product contract that macOS and user remain route/rate authority.
- Resample EQ: rejected category error. Parametric EQ coefficients already derive from output sample rate; no audio stream exists to resample inside EQ preparation.
